Что есть

  • Backend: NestJS 11, 93 модуля, 1156 эндпоинтов, 0 CVE, score 10/10
  • Frontend: Next.js 16 админка + Vite Telegram Mini App + маркетинг-сайт
  • Платежи: Payme, Click, Uzum, Telegram Stars
  • Фискал: Soliq.uz / Multikassa интеграция
  • Локализация: ru, uz, en
  • Predictive refill: EWMA + рекомендации, готов к работе
  • RBAC: 7 ролей, 0 нарушений canary-теста

Архитектурные решения

  • Хостинг — Railway (api/web/site/client/bot/Postgres/Redis), Vercel не используется
  • Multi-tenant через organizationId (114 entities)
  • Целевая модель ролей — 4 staff + 1 client + конструктор галочек (после стабилизации)
  • Resource-level доступ к машинам через machine-access (5 типов)
  • Owner = singleton — 1 пользователь (Jamshid)

Активные блокеры запуска

1. Railway web build пустой с 06.04.2026. Локальный docker build работает. Причина — биллинг или пауза проекта, не код.
2. Миграции БД не накатились на прод (отсутствуют 1776100000000+). Накатятся через releaseCommand при следующем deploy api.

5 действий до запуска (~2 часа)

  1. Прописать env-переменные в Railway (по ENV_VARIABLES.md)
  2. Добавить tzdata в Dockerfile API (Asia/Tashkent cron'ы)
  3. Подтвердить с бухгалтером доступ к inventory
  4. Включить resource-level права на машины
  5. Smoke-тест после деплоя — 10 сценариев
