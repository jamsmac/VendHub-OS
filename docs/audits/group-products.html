products · сами товары14 файлов · 33 эндпоинта

Самый нагруженный модуль. Хранит:

  • Готовые продукты (Капучино, Латте, Снек) — то что продаётся клиенту
  • Ингредиенты (зёрна, молоко, сахар, стаканы) — то из чего делается продукт
  • Рецепты — связь продукта с ингредиентами и их количеством
  • Цены — основная цена + промо-цены + историю изменений
  • Фото и описания

RBAC: Owner/Admin/Manager — все 33. Accountant/Warehouse — 13/33 (read + некоторые write по своим зонам). Operator — 10/33. Viewer — 13/33.

Связи

inventory (остатки), machines (что в каких слотах), orders (что покупают), categories (классификация), references (единицы измерения).

categories · иерархия категорий6 файлов · 5 эндпоинтов

Назначение: дерево категорий с parent_id. Используется для группировки и фильтрации в UI.

RBAC: Owner/Admin/Manager — все 5. Operator/Warehouse/Accountant — 2/5 (только чтение).

directories · пользовательские справочники16 файлов · 27 эндпоинтов

Назначение: механизм пользовательских справочников через MDM Directory Builder. Admin может создать новый справочник (например, «Допустимые причины возврата») с произвольной схемой полей.

Use case: добавление новых типов сущностей без изменения кода — всё через UI.

RBAC: Owner 20/27 (не все справочники нужны), Admin 27/27, Manager 21/27, Operator/Warehouse 8/27.

references · системные справочники17 файлов · 23 эндпоинта

Не путать с directories. References — это «жёсткие» системные справочники: валюты, единицы измерения, ставки НДС, страны. Меняются редко, через миграцию или сидинг.

RBAC: Owner/Admin — все, Manager 13/23, Accountant/Warehouse 13/23 (для финансовых и складских справочников). Operator/Viewer — 0 (не нужны).

custom-fields · extensibility5 файлов · 8 эндпоинтов

Назначение: добавление произвольных полей к существующим сущностям без миграции БД. Например, для Machine добавить «дата последней мойки», для Product — «диетические ограничения».

RBAC: Owner/Admin — управление, Manager 2/8 (только чтение).

Storage: JSONB поле в самой сущности с валидацией по Schema из таблицы custom_field_definitions.

favorites · избранное клиента7 файлов · 16 эндпоинтов

Назначение: кэширование любимых продуктов клиента в Mini App. «Купить как в прошлый раз», «Любимый кофе».

RBAC: 1/16 для всех ролей — это симметричный модуль, каждый видит свои favorites. Через JWT identity.

После Фазы 3.6: переедет в clients-scope, будет привязан к Client а не User.
