reports · главный отчётный модуль29 файлов · 3 controllers · 44 эндпоинта

3 sub-controller'а:

  • reports.controller.ts (26) — операционные отчёты (продажи, инвентарь, KPI)
  • controllers/analytics.controller.ts (6) — analytics-эндпоинты внутри модуля
  • controllers/vendhub-report.controller.ts (12) — VendHub-специфичные отчёты

Виды отчётов

  • Sales reports — выручка по периоду, машинам, продуктам
  • Inventory reports — остатки, оборачиваемость, списания
  • Operational reports — KPI операторов, маршруты, инциденты
  • Financial reports — для бухгалтера, налогов

Code smell: MIXED стиль @Roles в этом модуле — нужно унифицировать.

RBAC: Owner/Admin — все 44, Manager — 36/44, Accountant — 23/44 (финансовая часть), Viewer — 4/44 (только read).

analytics · KPI-дашборды5 файлов · 11 эндпоинтов

Назначение: KPI-виджеты для главного дашборда. Каждый виджет — отдельный эндпоинт с агрегированными данными.

Эндпоинты

  • GET /analytics/daily — daily stats для date range
  • POST /analytics/aggregate/:date — manual trigger агрегации
  • GET /analytics/dashboard — главный дашборд с виджетами и stats
  • POST/PATCH/DELETE /analytics/widgets — кастомизация дашборда

Code smell: 0 в OWNER колонке — Owner проходит через bypass, в декораторах не указан.

ai · AI-фичи13 файлов · 11 эндпоинтов

Назначение: AI-powered фичи через Anthropic Claude / OpenAI.

Use cases

  • AI Import — парсинг неструктурированных файлов (накладные в PDF) в структуру
  • Anomaly Detection — обнаружение аномальных продаж / расходов
  • Smart Suggestions — рекомендации по автоматизации операций
  • Natural language queries — «покажи мне топ-5 машин по выручке за неделю»

Зависимость: ANTHROPIC_API_KEY / OPENAI_API_KEY в env. Без ключей — фичи отключаются gracefully.

audit · audit log с 25+ типов событий16 файлов · 16 эндпоинтов

Назначение: append-only журнал всех значимых действий. Не путать с security модулем (тот про аномалии входа).

AuditAction enum (25+ значений)

CREATE, UPDATE, DELETE, SOFT_DELETE, RESTORE, LOGIN, LOGOUT, LOGIN_FAILED, PASSWORD_CHANGE, PASSWORD_RESET, PERMISSION_CHANGE, SETTINGS_CHANGE, EXPORT, IMPORT, BULK_UPDATE, BULK_DELETE, API_CALL, WEBHOOK_RECEIVED, PAYMENT_PROCESSED, REFUND_ISSUED, REPORT_GENERATED, NOTIFICATION_SENT, TASK_ASSIGNED, TASK_COMPLETED, MACHINE_STATUS_CHANGE, INVENTORY_ADJUSTMENT, FISCAL_OPERATION.

Limitation: mutable таблица. Теоретически Admin с прямым SQL может изменить логи. Решается в Фазе 4 — immutable storage.

RBAC: Owner 16/16, Admin 14/16 (свою орг), Manager 5/16 (только свои действия).
