▣Детали по модулям

organizations · сами организации12 файлов · 5 эндпоинтов

Главная сущность multi-tenancy. Каждая запись в системе имеет organizationId, который ссылается сюда.

Эндпоинтов мало (5) — потому что обычно одна организация. Создание новой — редкая операция (Owner-only).

RBAC: Owner 5/5, Admin 2/5 (только своя), Manager 1/5, остальные 1/5 (read общей инфы).

Связано с Решением 3 — организации остаются как фундамент multi-tenancy, не убираются.

contractors · поставщики и контракты12 файлов · 25 эндпоинтов · 2 controllers

Не путать с counterparty. Contractors = «поставщики товара/услуг» (кофейные зёрна, молоко, ремонт). Они закупают и поставляют.

2 controller'а

  • contractors.controller.ts — CRUD контрагентов
  • controllers/contract.controller.ts — контракты с ними (договора, условия)

RBAC: Owner/Admin 25/25, Manager 20/25, Accountant 14/25 (для финансовых отношений).

counterparty · контрагенты для финрасчётов5 файлов · 12 эндпоинтов

Назначение: юридические сущности для финансовых расчётов. Один contractor может быть связан с несколькими counterparties (если у поставщика несколько юрлиц).

Use case: для бухгалтерии важна не «организация-поставщик», а конкретное юрлицо для НДС, отчётности.

RBAC: Owner/Admin 12/12, Manager 9/12, Accountant 6/12. Operator/Warehouse — 0 (это финансовая зона).

settings · настройки организации7 файлов · 11 эндпоинтов

Назначение: per-org настройки: валюта, часовой пояс, локаль, рабочие часы, политики возвратов.

RBAC: Owner 10/11, Admin 6/11. Manager 0/11 — настройки только для верхнего уровня.

cms · контент-менеджмент9 файлов · 11 эндпоинтов · 2 controllers

!Возможный refactoring

В аудите была отмечена потенциальная избыточность между organizations / contractors / counterparty. Действительно ли все три нужны?

СущностьЧем уникальна
organizationsТвоя компания или франчайзи. Главная единица tenancy. organizationId везде.
contractorsПодрядчики/поставщики (тот кто привозит товар, обслуживает технику).
counterpartyЮр. лица для финансовых расчётов (один contractor может иметь несколько counterparty — для разных видов услуг и НДС).
Все три различимы и нужны. Нет необходимости консолидировать. Это была техническая проверка по результатам обсуждения «может организации лишние?» — вердикт оставлен в ADR-003.
