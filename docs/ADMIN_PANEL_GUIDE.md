## Роли и доступ

Система поддерживает **7 ролей** (от высшей к низшей):

| Роль | Что видит | Типичный пользователь |
|------|-----------|----------------------|
| **owner** | Всё | Владелец бизнеса |
| **admin** | Всё кроме мульти-орг | Директор филиала |
| **manager** | Операции + Маркетинг + HR + Отчёты | Управляющий |
| **operator** | Машины + Задачи + Поездки | Техник на маршруте |
| **warehouse** | Склад + Инвентарь + Продукты | Кладовщик |
| **accountant** | Финансы + Заказы + Отчёты | Бухгалтер |
| **viewer** | Только просмотр дашборда | Наблюдатель |

## 2. МАШИНЫ

**Путь:** `/dashboard/machines`
**Доступ:** owner, admin, manager, operator
**Связан с:** Контейнеры, Задачи, Поездки, Обслуживание, Локации, Инвентарь

### Общий вид
```
+--------------------------------------------------+
| Всего: 31 | Активных: 28 | Внимание: 3 | Ошибки: 0|
+--------------------------------------------------+
| [+ Новая машина]  Поиск: [________]               |
| Фильтр: [Все статусы v]  Вид: [Список] [Карта]   |
+--------------------------------------------------+
| +------------+ +------------+ +------------+      |
| | VM-0001    | | VM-0002    | | VM-0003    |      |
| | Americano  | | Cappuccino | | Latte      |      |
| | ул. Навои  | | ТЦ Samarkand| | ТАТУ      |      |
| | Остаток 78%| | Остаток 34%| | Остаток 91%|      |
| | Касса: 450K| | Касса: 120K| | Касса: 680K|      |
| | [Активна]  | | [Внимание] | | [Активна]  |      |
| +------------+ +------------+ +------------+      |
+--------------------------------------------------+
```

### Сценарии действий

**Сценарий 1: Просмотр списка машин**
```
Список машин → Поиск по имени → Фильтр по статусу → Переключение Список/Карта
```

**Сценарий 2: Создание новой машины**
```
Кнопка [+ Новая машина] → Форма (Имя, Код, Серийный №, Тип, Производитель)
→ Сохранить → Появляется в списке
```

**Сценарий 3: Детальный просмотр машины (8 вкладок)**
```
Клик на машину → Страница детали с 8 вкладками:

  ┌─────────┬──────────┬───────────┬───────────┐
  │ Паспорт │ Содержим.│ Аналитика │ Инкассация│
  ├─────────┼──────────┼───────────┼───────────┤
  │ Обслуж. │ Локации  │  Задачи   │ Хронология│
  └─────────┴──────────┴───────────┴───────────┘

  • Паспорт — спецификации машины
  • Содержимое — бункеры, компоненты, чистка
  • Аналитика — P&L, продажи, динамика
  • Инкассация — история сборов наличных
  • Обслуживание — записи ТО
  • Локации — где стоит/стояла
  • Задачи — связанные задачи
  • Хронология — лента активности
```

**Сценарий 4: Быстрые действия из детали**
```
Шапка машины → [Инкассация] [Загрузка] [Добавить задачу]
```

### Статус: Полностью работает
- CRUD, поиск, фильтры, карта, детали с 8 вкладками

### Что доработать
- Реальная телеметрия в реальном времени (WebSocket)
- Кнопка "Загрузка" — пока без формы пополнения

## 6. ЗАДАЧИ

**Путь:** `/dashboard/tasks`
**Доступ:** owner, admin, manager, operator
**Связан с:** Машины, Поездки, Сотрудники

### Два режима отображения
```
+--------------------------------------------------+
| Всего: 156 | Ожидающие: 12 | В работе: 8 | Просрочено: 2|
+--------------------------------------------------+
| [+ Новая задача]  Вид: [Список] [Канбан]         |
+--------------------------------------------------+

Режим КАНБАН:
┌──────────┬──────────┬──────────┬──────────┐
│ Ожидает  │ Назначена│ В работе │ Готово   │
│   (12)   │   (5)    │   (8)    │  (131)   │
├──────────┼──────────┼──────────┼──────────┤
│ Заправка │ Ремонт   │ Чистка   │ Инкассац.│
│ VM-0005  │ VM-0012  │ VM-0003  │ VM-0001  │

└──────────┴──────────┴──────────┴──────────┘
```

### Типы задач
- Заправка (refill)
- Инкассация (collection)
- Чистка (cleaning)
- Ремонт (repair)
- Установка (install)
- Демонтаж (removal)
- Аудит (audit)
- Инспекция (inspection)

### Сценарии
```
Менеджер: [+ Задача] → Тип → Приоритет → Дедлайн → Назначить → Создать
Оператор: Мои задачи → [Начать] → Фото до → Выполнить → Фото после → [Завершить]
Менеджер: Канбан → Видит прогресс → Переназначить / Отложить / Отклонить
```

### Статус: Полностью работает

## 7. КОЛЛЕКЦИИ (Инкассация)

**Путь:** `/dashboard/collections`
**Доступ:** owner, admin, manager, accountant
**Связан с:** Машины, Транзакции, Поездки, Финансы

### Двухэтапный процесс
```
ЭТАП 1: Оператор собирает наличные из машины
  Оператор → Машина → Считает деньги → Создаёт запись → Статус: "Собрано"

ЭТАП 2: Бухгалтер принимает и сверяет
  Бухгалтер → Видит "Собрано" → Проверяет сумму → [Принять] / [Отклонить]
  → Вводит фактическую сумму → Статус: "Принято"
```

### Что показывает
```
+--------------------------------------------------+
| Всего: 340 | Общая сумма: 34M UZS | Ожидают: 5  |
| Сегодня: 2,100,000 UZS                           |
+--------------------------------------------------+
| Фильтр: [Все / Собрано / Принято / Отменено]     |
+--------------------------------------------------+
| Статус   | Машина  | Оператор | Сумма    | Дата   |
|----------|---------|----------|----------|--------|
| Собрано  | VM-0005 | Азиз     | 450,000  | Сегодня|
| Принято  | VM-0012 | Бобур    | 680,000  | Вчера  |
+--------------------------------------------------+
```

### Статус: Работает

**Связан с:** Маршруты, Задачи, Машины, Транспорт, Аналитика поездок

```
Запланирована → Активна → Завершена
                  ↓
              Отменена

Менеджер: Создаёт маршрут → Оператор: Начинает поездку

```

### Под-страницы
- `/trips` — Список поездок
- `/trips/[id]` — Детали (маршрут, остановки, задачи, аномалии)
- `/trips/tracker` — Трекер в реальном времени
- `/trips/analytics` — Аналитика по поездкам

### Сценарии
```
Оператор: [Начать поездку] → Выбрать маршрут → Транспорт → Старт
→ Посещает точки → Выполняет задачи → [Завершить]

Менеджер: Видит активные поездки → Трекер → Аномалии → Аналитика
```

## 14. ФИНАНСЫ

**Путь:** `/dashboard/finance`
**Доступ:** owner, admin, accountant
**Связан с:** Транзакции, Платежи, Инкассации, Сверка, Фискализация

### 10 вкладок
```
┌─────────┬──────┬──────────┬───────────┬─────────┐
│ Обзор   │ P&L  │ Денеж.   │ Транзакции│ Счета   │
│         │      │ поток    │           │         │
├─────────┼──────┼──────────┼───────────┼─────────┤
│ Платежи │Сверка│ Фискализ.│  Бюджет   │ Отчёты  │
└─────────┴──────┴──────────┴───────────┴─────────┘
```

**Обзор:** Выручка, расходы, прибыль, графики
**P&L:** Выручка → Себестоимость → Валовая прибыль → OpEx → EBITDA
**Денежный поток:** Операционный / Инвестиционный / Кумулятивный
**Транзакции:** Фильтр по методу оплаты, поиск, пагинация
**Счета:** Входящие/исходящие, фильтр по типу и статусу
**Платежи:** Payme, Click, Uzum, HUMO, UZCARD — балансы и статус
**Сверка:** Расхождения между аппаратными и программными данными
**Фискализация:** Чеки Multikassa, суммы НДС
**Бюджет:** План vs Факт по категориям
**Отчёты:** Финансовые отчёты

### Статус: Частично работает
- Обзор и транзакции — реальные данные
- P&L, денежный поток, бюджет — **моковые данные** (нужна доработка)

```

              ┌─────────────────┼─────────────────┐

              v                v                  v
         Журнал работ    Баланс дней          Оценки

```

  │
  ├──> Остановка 2: Машина VM-0012
  │    └── Задача: Инкассация → Собирает наличные → Записывает сумму
  │
  ├──> Остановка 3: Машина VM-0003
  │    └── Задача: Ремонт → Описание → Запчасти → Завершение
  │
  v
Завершает поездку → Аналитика поездки (км, время, задачи)
  │
  v
Бухгалтер принимает инкассацию → Сверяет → Финансы

## Навигация по ролям (что видит каждый)

### Оператор (техник на маршруте)
```
Дашборд → Машины → Задачи → Поездки → Обслуживание
→ Контейнеры → Оборудование → Алерты → Инциденты
→ Транспорт → Карта → Уведомления → Помощь
```
**Типичный день:** Смотрит задачи → Начинает поездку → Обслуживает машины → Завершает

### Бухгалтер
```
Дашборд → Заказы → Инкассации → Транзакции → Платежи
→ Финансы → Фискализация → Сверка → Отчёты платежей
→ Начальные остатки → Контрагенты → Отчёты → Уведомления
```
**Типичный день:** Принимает инкассации → Сверяет платежи → Формирует отчёты

### Менеджер
```
Всё что видит оператор и бухгалтер +
Продукты → Инвентарь → Маршруты → Рейтинги
→ Акции → Инвестор → Лояльность → Жалобы
→ Сотрудники → Команда → Журнал работ
→ Склад → Локации → Сайт → Аналитика
→ Справочники → Доступ к машинам
```
**Типичный день:** Дашборд → Проверяет алерты → Создаёт задачи → Контролирует операторов → Отчёты

### Владелец (owner)
```
Всё что видит менеджер +
Организации → Пользователи → Приглашения
→ Импорт → Интеграции → Вебхуки → Аудит
→ Справочные данные → API документация → Настройки
```
