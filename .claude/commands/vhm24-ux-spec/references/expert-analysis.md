# VendHub Expert Analysis Framework

## Expert Personas

### 1. Senior UX Designer (15+ лет опыта)
**Фокус**: Пользовательский опыт, информационная архитектура, пользовательские потоки

**Анализирует**:
- Как пользователь достигает цели?
- Сколько шагов до результата?
- Где пользователь может запутаться?
- Какие действия критичны для бизнеса?
- Какие ошибки может совершить пользователь?

**Вопросы для спецификации**:
```
□ Кто основной пользователь этого экрана?
□ Какую задачу он решает?
□ Какой путь пользователя до этого экрана?
□ Куда пользователь переходит после?
□ Какие действия должны быть очевидны?
□ Какие действия должны требовать подтверждения?
```

### 2. UI Designer (10+ лет опыта)
**Фокус**: Визуальная иерархия, консистентность, брендинг

**Анализирует**:
- Соответствует ли дизайн-системе "Warm Brew"?
- Правильная ли визуальная иерархия?
- Консистентны ли компоненты?
- Достаточен ли контраст для читаемости?
- Работает ли дизайн в dark mode?

**Вопросы для спецификации**:
```
□ Какие компоненты из дизайн-системы использовать?
□ Какая визуальная иерархия элементов?
□ Какие цветовые акценты для статусов?
□ Какие иконки передают смысл действий?
□ Как выглядит пустое состояние?
□ Как выглядит состояние загрузки?
```

### 3. Interface Specialist (8+ лет опыта)
**Фокус**: Взаимодействия, микроанимации, accessibility

**Анализирует**:
- Какие hover/active/focus состояния?
- Нужны ли анимации переходов?
- Доступен ли интерфейс для screen readers?
- Работает ли с клавиатурой?
- Правильные ли touch targets для мобильных?

**Вопросы для спецификации**:
```
□ Какие интерактивные элементы на экране?
□ Какие состояния у каждого элемента?
□ Какие модальные окна/drawers открываются?
□ Какие уведомления показываются?
□ Какие keyboard shortcuts нужны?
```

### 4. Frontend Architect
**Фокус**: Технические паттерны, производительность, переиспользуемость

**Анализирует**:
- Какие компоненты можно переиспользовать?
- Какие данные нужны от API?
- Какое состояние хранить локально vs на сервере?
- Какие оптимизации для больших списков?

**Вопросы для спецификации**:
```
□ Какие tRPC endpoints нужны?
□ Какие Zustand stores используются?
□ Какие данные кешировать?
□ Нужна ли пагинация/виртуализация?
□ Какие типы TypeScript определить?
```

## Чеклист качества спецификации

Перед генерацией кода проверить:

### UX ✓
- [ ] Понятна цель экрана
- [ ] Определён пользовательский путь
- [ ] Описаны все действия пользователя
- [ ] Предусмотрены ошибочные сценарии

### UI ✓
- [ ] Указаны все компоненты из дизайн-системы
- [ ] Определена визуальная иерархия
- [ ] Описаны цветовые акценты
- [ ] Предусмотрен dark mode

### Interface ✓
- [ ] Описаны все интерактивные состояния
- [ ] Указаны модальные окна
- [ ] Определены уведомления
- [ ] Предусмотрена accessibility

### Tech ✓
- [ ] Определены API endpoints
- [ ] Описана структура данных
- [ ] Указаны TypeScript типы
- [ ] Продуманы оптимизации
